# Boss Rush Game — 設計文件

一款 **即時動作 Boss Rush** 遊戲:玩家連續挑戰 20 關,沿途靠隨機抽選累積技能 build,並透過生活系統(採集 / 耕種 / 釣魚)打造永久裝備與補血道具。

---

## 設計支柱(Design Pillars)

| 支柱 | 內容 |
|---|---|
| **即時動作戰鬥** | 玩家即時移動、閃避、攻擊;戰鬥迴圈跑在前端 Canvas |
| **單局策略層** | 技能 build —— 每局隨機抽選、累積、升級,Roguelike 重玩價值 |
| **永久養成層** | 裝備 —— 靠生活系統打造與強化,跨整個遊戲保留 |
| **資源循環** | 採集 / 耕種 / 釣魚 → 材料 → 補血道具 + 裝備 + 強化 |

### 兩條「角色強度」軸(核心觀念)

|  | 技能 build | 裝備 |
|---|---|---|
| 保留範圍 | **單局**(重來就洗掉) | **永久**(跨遊戲保留) |
| 來源 | 關卡中隨機抽選 | 生活系統打造 + Boss 掉落 |
| 性質 | 隨機性、每局不同 | 長期養成、越玩越強 |

兩者互補:輸了一局也有「材料 / 裝備變強」的收穫,降低純挫敗感。

---

## 技術棧

| 層 | 技術 |
|---|---|
| 前端 | React(UI/選單)+ **Pixi.js**(Canvas 戰鬥/場景渲染)+ **Framer Motion**(UI 動畫)+ Tailwind + Zustand |
| 後端 | Java + Spring Boot(REST API) |
| 資料庫 | PostgreSQL(技能/裝備設定用 `jsonb`) |
| 美術素材 | PixelLab AI(API 整合,key 存 `PIXELLAB_API_KEY`)+ 程式粒子特效 |
| 視角 | 俯視角(top-down) |
| 目標平台 | 網頁版 + 手機版(同一套程式與素材,畫面縮放;手機鎖橫向 + 虛擬搖桿) |

**戰鬥權威性**:即時動作採 **前端主導(client-authoritative)**,後端負責存檔、屬性計算與基本合理性檢查,第一版不做嚴格反作弊。

## 開發順序(尚未開工)

每階段都產出**可執行、可驗收**的成果,測過通過才進下一階段。詳見 [development-plan.md](development-plan.md)。

| 階段 | 成果 |
|---|---|
| 0 | 環境與骨架(三層連通) |
| 1 | 角色移動(Canvas 核心 + 跨平台縮放) |

| 3 | 關卡串接 + 技能抽選 + 進度存檔 |
| 4 | 家園 hub + 生活系統(產材料) |
| 5 | 合成 + 補血道具 + 裝備(經濟循環) |
| 6 | 美術整合 + UI 打磨 + 平衡 |

> 狀態:**設計階段**。本資料夾僅含設計文件,尚未建立任何程式碼。

# 開發階段流程(可逐階段驗收)

原則:**每個階段都產出能實際執行、可測試的成果**;你親自測過、通過「通過標準」後,才進下一階段。每階段盡量是**垂直切片**(前後端打通一小塊),而不是一次做完某一層。

## 運作方式(我們的合作節奏)

```
我實作一個階段
   → 我跑起來 / 給你明確測試步驟
   → 你測試
   → 通過 ✅ 進下一階段 / 不通過 ❌ 我修正後再測
```

> 隨時可在某階段插入調整;不通過不前進。

---

## 階段 0:環境與骨架

**目標**:三層(React / Spring Boot / Postgres)能互相連通。

**內容**:建立專案結構、React 前端、Spring Boot 後端、Postgres(Docker Compose);一條 `GET /api/health` 打通到前端顯示。

**你可以測什麼**
1. 啟動後端 → 瀏覽器開 `/api/health` 回 `ok`。
2. 啟動前端 → 頁面顯示「後端連線正常」(實際打 API 拿到的)。
3. 後端確實連上 Postgres(啟動無錯、能建表)。

**通過標準**:前端 ↔ 後端 ↔ 資料庫 一條線通,無錯誤。

---

## 階段 1:角色移動(Canvas 核心)

**目標**:可走動的俯視角色 + 正確的跨平台縮放。

**內容**:`<CanvasStage>` 邏輯解析度 480×270 + 整數縮放 / letterbox / 鄰近取樣;載入玩家 sprite;鍵盤 + 觸控搖桿移動。

**你可以測什麼**
1. 桌機用 WASD / 方向鍵移動角色,流暢。
2. 視窗縮放 / 手機橫向:畫面正確縮放、像素不糊、有 letterbox。
3. 手機虛擬搖桿能移動。

**通過標準**:角色移動順暢,桌機與手機畫面都正確。

---

## 階段 2:戰鬥垂直切片(一關)

**目標**:單一關卡能打、能贏、能死。

**內容**:敵人生成 + 簡單 AI;1 個主動技能**自動施放 + 自動瞄準**;玩家 / 敵人 HP;閃避(無敵幀);清場過關 / 死亡判定。

**你可以測什麼**
1. 進入戰鬥,技能自動朝最近敵人施放。
2. 打倒所有敵人 → 顯示過關。
3. 被打到會扣血、歸零會死亡。
4. 閃避時短暫無敵。

**通過標準**:單場戰鬥可玩、有輸有贏的手感。

---

## 階段 3:關卡串接 + 技能抽選 + 進度

**目標**:完整一局 boss rush 流程跑通(內容可先用佔位)。

**內容**:20 關流程;第 5/10/15/20 關為 Boss;每關之間 3 選 1 抽選、累積 build;進度存後端 `run_progress`。

**你可以測什麼**
1. 連續闖關,每關之間出現 3 選 1,build 會累積 / 升級。
2. 第 5/10/15 關出現小 Boss、第 20 關大 Boss。
3. 中途重整頁面 → 進度仍在(已存後端)。

**通過標準**:從開局到通關 / 戰敗的整個單局循環可運作。

---

## 階段 4:家園 hub + 生活系統

**目標**:可走動家園 + 能生產材料。

**內容**:家園 Canvas 場景 + 設施互動;農田(真實時間成長)、採集、釣魚;背包 / 倉庫。

**你可以測什麼**
1. 在家園走動,靠近設施出現互動提示。
2. 播種 → 等待(可加除錯快轉)→ 收成。
3. 採集 / 釣魚拿到材料,背包數量正確。
4. 離線後回來,作物依真實時間成長(後端判定)。

**通過標準**:能透過生活活動穩定產出材料。

---

## 階段 5:合成 + 補血道具 + 裝備

**目標**:打通「生活 → 養成 → 戰鬥」經濟循環。

**內容**:合成配方 → 補血道具 + 裝備;裝備 loadout / 強化;補血道具可在戰鬥中使用;**裝備屬性由後端彙整**。

**你可以測什麼**
1. 用材料合成補血道具,攜帶進關,戰鬥中使用回血。
2. 合成 / 裝備武器 → 角色屬性改變、傷害變高。
3. 強化裝備 `+1…`,屬性隨之提升。

**通過標準**:材料 → 合成 → 裝備 / 道具 → 影響戰鬥,整條循環成立。

---

## 階段 6:美術整合 + UI 打磨 + 平衡

**目標**:成為一款風格一致、好玩的完整版本。

**內容**:接入生成素材(角色 / 敵人 / 技能 / tileset / UI);真實技能圖示;選角畫面;手機觸控定稿;數值平衡。

**你可以測什麼**
1. 桌機 + 手機各完整跑一輪。
2. 美術正確顯示、風格一致、UI 在手機上清楚。
3. 難度曲線合理(不會太簡單 / 太硬)。

**通過標準**:完整可玩、視覺一致、雙平台體驗良好。

---

## 測試方式補充

- **手動遊玩測試**為主(每階段我會把它跑起來 / 給你步驟)。
- 後端關鍵邏輯(屬性彙整、合成、進度)可加**自動化測試**輔助。
- 每階段保留「除錯捷徑」(如快轉作物、跳關)方便你快速驗收。

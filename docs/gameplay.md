# 玩法與關卡設計

## 地圖系統(10 張地圖 × 每張 20 關)

遊戲分 **10 張地圖(世界 / 章節)**,每張都是一輪完整的 20 關 boss rush,各有**主題、tileset、敵人、難度**。共 10 × 20 = 200 關。

- **資料驅動**:地圖與關卡都是資料;一次設計好系統,加地圖只是新增資料。
- **第一版只做地圖 1**(見下),其餘 2–10 之後依同結構擴充。

### 每張地圖的關卡結構(20 關)

連續挑戰,關與關之間進入「抽選技能」畫面,Boss 關後可獲得較佳獎勵。

| 關卡 | 類型 | 說明 |
|---|---|---|
| 1–4 | 普通怪 | 入門,建立 build 基礎 |
| **5** | **小 Boss** | 第一個門檻 |
| 6–9 | 普通怪 | 難度遞增 |
| **10** | **小 Boss** | 中段檢驗 |
| 11–14 | 普通怪 | |
| **15** | **小 Boss** | 終戰前最後試煉 |
| 16–19 | 普通怪 | 高強度 |
| **20** | **大 Boss(最終)** | 多階段、地圖最終挑戰 |

- 每張地圖:普通關 16、小 Boss 3(5/10/15)、大 Boss 1(20)。

### 地圖進度

- **依序解鎖**:首次通關地圖 N(過第 20 關)→ 解鎖地圖 N+1。
- **可重刷**:**只要該地圖通關過一次,即可無限重刷**(farming 裝備掉落與材料)。
- **難度遞增**:後面的地圖敵人更強(用地圖難度係數調 `enemy_config`)。
- **build 每局重置**(進任一地圖都是新的一局 build);**裝備為永久**,跨地圖保留。
- **核心循環**:越後面地圖越硬 → 重刷舊地圖 + 生活系統養裝備 → 變強再推進新地圖。裝備取得見[下方掉落](#裝備掉落取得)與 [equipment.md](equipment.md)。

### 裝備掉落(取得)

- **主要來源:通關掉落(機率)**。擊敗 Boss(5 / 10 / 15 / 20)各 roll 一次掉落,**最終 Boss(20)掉落最好**;稀有度依機率權重 roll(普通→傳說)。
- **地圖難度越高 → loot table 越好**(更高機率掉高稀有度 / 高基礎數值)。
- **重刷舊地圖**正是為了刷裝備。
- 掉落由**後端 roll**(防作弊)。
- 次要來源:生活系統合成(強化材料、補血道具、少數特定裝備),見 [life-system.md](life-system.md)。

## 單局流程(一次 Boss Rush)

```
開局
 └ 從隨機 4 個技能中選 2 個(起手 build)
     │
     ├─ 第 1 關(普通怪)→ 清關
     │     └ 抽選:隨機 3 選 1(新技能 或 升級現有技能)
     ├─ 第 2~4 關 …(同上)
     │
     ├─ 第 5 關(小 Boss)→ 清關
     │     └ 強化抽選:稀有技能 / 必為升級 三選一
     │
     ├─ … 重複至第 19 關 …
     │
     └─ 第 20 關(大 Boss)→ 通關!
```

- **死亡**:整局結束(可設定:回家園,保留本局採集到的材料 → 強化裝備,再挑戰)。
- 過第 20 關約持有 **5~8 個技能、各有等級** 的成形 build。

---

## 抽選機制(技能 draft)

- 技能池總量目標 ~50 種,但**每次只隨機呈現 3~4 個**,避免選擇癱瘓(Roguelike 標準做法)。
- 每關之間 3 選 1:可選「新技能加入 build」或「升級已有技能(等級 +1)」。
- 小 Boss 關(5/10/15)給更好的選項池(稀有 / 保證升級)。
- `run_progress` 記錄該關抽選**種子**,避免玩家刷新重抽。

## 角色

新遊戲開始時可選擇**男 / 女**角色,**純外觀差異、不影響數值或玩法**(公平、受眾更廣)。之後可擴充髮型 / 配色等自訂。

---

## 控制模型

主動技能**自動施放 + 自動瞄準**;玩家只手動控制**移動、閃避(dash)、使用補血道具**。操作重點在走位與閃避,深度來自 build 搭配。詳見 [skills.md 施放模型](skills.md)。

---

## 補血機制

- 戰鬥中**不靠技能回滿血**(避免架空生活系統)。
- 回血來自 **生活系統合成的補血道具**,關卡內手動使用(見 [life-system.md](life-system.md))。
- 技能的 sustain 走條件式(吸血需命中、瀕死無敵等),不能取代補血道具。

---

## 敵人設計重點

- 每個 Boss 需有**可辨識、可學習**的攻擊模式(telegraph → 玩家反應)。
- Boss 血量到門檻時觸發**階段轉換**(換招 / 加速 / 變形)。
- 普通關以小群雜兵 + 簡單彈幕為主,鋪墊 Boss 關張力。
- 敵人參數(HP / 攻擊 / 攻擊模式 ID)存於 `stage` 表,資料驅動。

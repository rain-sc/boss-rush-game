# 前端 UI 設計

React(選單 / 面板)+ Canvas(可走動場景:家園 hub 與戰鬥)混合架構,手機優先、響應式。

---

## 1. 技術選型

| 項目 | 選擇 | 原因 |
|---|---|---|
| 畫面切換 | **screen state machine**(Zustand 管理),非 URL 路由 | 遊戲用狀態切畫面比網址直覺 |
| 全域狀態 | **Zustand** | 輕量;存 player / 背包 / 裝備 / 進度 |
| 樣式 | **Tailwind CSS** + 自訂像素主題 | 響應式快、易維護 |
| 中文像素字 | **Zpix(最像素)/ Cubic 11** | 中文也要像素風才統一 |
| 面板 / 邊框 | 9-slice 像素邊框(PixelLab UI 素材) | 風格一致 |
| 可走動場景 | **Canvas 遊戲迴圈**(家園 + 戰鬥共用) | 引擎重用 |
| 場景內 HUD | 畫在 Canvas | 跟像素縮放一致 |
| 設施 / 選單面板 | **React DOM 疊層**(CSS 響應式) | 文字清楚、版面自動重排 |

> **關鍵架構**:家園 hub 與戰鬥都是 Canvas 場景,共用「移動 + 碰撞 + 渲染」核心;差別只在場景內容(設施 vs 敵人)。設施互動、背包、合成等**面板用 DOM 疊在 Canvas 上**。

---

## 2. 畫面地圖與導覽流程

```
主選單
 ├ 新遊戲 → 角色選擇(男 / 女,純外觀)→ 家園
 └ 家園 hub(可走動 Canvas 場景)
     │  走到設施旁按互動 → 彈出 DOM 面板:
     │   ├ 農場(播種 / 收成)
     │   ├ 池塘(釣魚)
     │   ├ 採集點(採集)
     │   └ 房子(合成 / 裝備 / 背包)
     └ 走到傳送門 → 地圖選擇(10 章節,已通關可重刷)
          → 技能起手(4 選 2)
          → 戰鬥場景 ⇄ 技能抽選(每關之間 3 選 1)
          → 結算(通關 / 戰敗 + Boss 裝備掉落)
          → 回家園
```

## 3. 各畫面設計

### 主選單
```
            BOSS RUSH
        ┌──────────────┐
        │  ▶ 繼續遊戲   │
        │  ＋ 新遊戲    │
        │  ⚙ 設定       │
        └──────────────┘
```

### 角色選擇(新遊戲)
```
        ── 選擇角色 ──
   ┌──────────┐   ┌──────────┐
   │   🧑 男   │   │   👩 女   │
   │ [男角預覽]│   │ [女角預覽]│
   └──────────┘   └──────────┘
         [ 確定，進入家園 ]
```
- 純外觀,不影響數值;之後可擴充髮型 / 配色。

### 家園 hub(可走動 Canvas)
```
┌─────────────────────────────────────────┐
│  🌳     🏠 房子(合成/裝備/背包)   🌳     │
│                                           │
│      🧑 ← 玩家走動                          │
│  🌾🌾 農田      🪨 採集點      🎣 池塘      │
│  ⊕搖桿                      [互動]  [▶出發]│
└─────────────────────────────────────────┘
```
- 走近設施 → 顯示「按互動」提示;互動 → 彈出對應 DOM 面板。
- 桌機 WASD/方向鍵移動、`E` 互動;手機虛擬搖桿 + 互動鈕。

### 設施面板(DOM 疊層,範例:農場)
```
┌─ 農場 ───────────────────────[X]┐
│ [🌱][🌱][🥕][空][空][空]          │
│  ▸ 播種    ▸ 收成                  │
│  種子:蘿蔔 x5  小麥 x2             │
└───────────────────────────────────┘
```
其餘面板:**釣魚**(互動 / 隨機)、**採集**(冷卻)、**合成**(配方列表 → 產出)、**裝備**、**背包**。

### 裝備面板
```
┌─ 裝備 ────────────┬─ 角色屬性 ──────┐
│ 武器 [炎之劍 +4]  │ 攻擊  142        │
│ 防具 [鐵甲   +2]  │ 生命  580        │
│ 飾品 [   空   ]   │ 暴擊  18%        │
│ [強化][分解][更換]│ 火傷 +12%        │
└───────────────────┴──────────────────┘
```

### 技能起手(4 選 2)/ 抽選(3 選 1)
```
          ── 選擇一項強化 ──
 ┌────────┐  ┌────────┐  ┌────────┐
 │ [火球] │  │ [冰錐] │  │ [疾風步]│
 │ 火·主動 │  │ 冰·新! │  │ 風·升級 │
 │ 傷害+8  │  │ 命中緩速│  │ Lv2→3  │
 └────────┘  └────────┘  └────────┘
```

### 戰鬥 HUD(畫在 Canvas,手機橫向)

主動技能**自動施放**(見 [skills.md 施放模型](skills.md)),所以**沒有技能按鈕**;技能冷卻只是**顯示用指示器**。玩家只控制移動、閃避、補血。
```
┌─────────────────────────────────────────┐
│ LV7   BOSS ████████░░ 80%                 │
│                                           │
│             [ 戰鬥畫面 ]                   │
│  ⊕                                ◉ 閃避   │
│ 搖桿                              🧪x3 使用 │
│ HP ██████░░ 60%   🔥3.2s ❄️就緒 ⚡1.1s     │
└─────────────────────────────────────────┘
```
HUD 元素:關卡 / Boss 血條、玩家 HP、**技能冷卻指示器(僅顯示)**、補血道具數量 + 使用鈕、閃避鈕。
手機觸控只需:**搖桿 + 閃避鈕 + 補血鈕**。

### 結算(通關 / 戰敗)
```
┌─ 結算 ───────────────────┐
│ 到達:第 12 關            │
│ 獲得材料:木x4 礦x2 草x6  │
│ 掉落:[鐵劍]              │
│      [ 回家園 ]          │
└──────────────────────────┘
```

## 4. 響應式 / 手機

- **橫向鎖定**;Canvas 場景照 [assets.md §8](assets.md) 整數縮放 + letterbox。
- 觸控:家園與戰鬥共用**虛擬搖桿**;戰鬥只需閃避 + 補血鈕(技能自動施放)、家園用互動鈕(≥44px)。
- DOM 面板用 CSS grid 自動重排,小螢幕換行;圖示維持原生像素尺寸。
- 避開瀏海安全區(`safe-area-inset`)。

---

## 5. 視覺風格

- 像素字(Zpix / Cubic 11)、9-slice 邊框、與 [assets.md §1](assets.md) 同一套調色盤。
- 面板半透明深色底 + 像素外框;按鈕有 hover / press 狀態。
- 動態回饋(受傷震屏、拾取彈跳)走程式特效,不靠 AI 素材。

---

## 6. 可重用 React 元件(初步)

| 元件 | 用途 |
|---|---|
| `<Panel>` | 9-slice 面板容器(設施 / 選單共用) |
| `<SkillCard>` | 技能卡(起手 / 抽選 / build 顯示) |
| `<ItemSlot>` | 道具 / 裝備格(背包 / 合成 / 裝備) |
| `<StatList>` | 屬性列表(裝備 / 角色) |
| `<TouchControls>` | 虛擬搖桿 + 按鈕(手機) |
| `<CanvasStage>` | 承載 Canvas 場景並處理縮放 / DPR / letterbox |

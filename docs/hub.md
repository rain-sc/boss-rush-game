# 家園(Hub)設計

可走動的俯視角據點,是「生活 ↔ 戰鬥」循環的中心:在此採集、耕種、釣魚、合成、管理裝備,並從傳送門出發挑戰 boss rush。家園也是**存檔安全點**。

> 家園是 Canvas 場景,與戰鬥共用移動 / 渲染核心(見 [architecture.md](architecture.md));設施互動彈出 React DOM 面板(見 [ui.md](ui.md))。

## 設施

| 設施 | 互動面板(DOM) |
|---|---|
| 🏠 房子 | 合成、裝備管理、背包 / 倉庫、存檔 |
| 🌾 農田 | 播種 / 收成(多格) |
| 🎣 池塘 | 釣魚 |
| 🌲 採集林 | 採集木材 / 草藥 |
| ⛏️ 礦點 | 採集礦石 |
| 🌀 傳送門 | 開啟**地圖選擇**(10 章節)→ 出發挑戰 boss rush |

**互動方式**:走近設施 → 顯示「按互動」提示 → `E` / 互動鈕 → 彈出對應面板。

### 傳送門 → 地圖選擇

互動傳送門彈出 10 張地圖列表(見 [gameplay.md 地圖系統](gameplay.md)):

```
┌─ 選擇地圖 ──────────────────────────[X]┐
│ ✅ 1 翠綠林地   ✅ 2 幽暗洞窟  🔒 3 灼熱沙漠 │
│ 🔒 4 迷霧沼澤   🔒 5 凜冬雪原  🔒 6 古代遺跡 │
│ 🔒 7 熔岩火山   🔒 8 雷鳴高塔  🔒 9 腐朽墓園 │
│ 🔒 10 虛空深淵                              │
│  ✅=已通關(可重刷)  ▶=可挑戰  🔒=未解鎖    │
│            [ ▶ 進入選定地圖 ]               │
└────────────────────────────────────────────┘
```
- **依序解鎖**:首次通關地圖 N 才解鎖 N+1。
- **已通關地圖可無限重刷**(刷裝備掉落 / 材料)。
- 選定地圖 → 技能起手(4 選 2)→ 進入該地圖第 1 關。

## 時間模型:真實時間(real-time)

作物成長與採集點復原**依真實時間推進,不依賴是否出戰**(適合手機 / 零散遊玩)。

| 對象 | 機制 |
|---|---|
| 作物 | 播種 → `ready_at = now + 成長時間`;`now ≥ ready_at` 即可收成 |
| 採集點(林 / 礦) | 採集後進入冷卻 → `cooldown_until = now + 復原時間`;冷卻中顯示「枯竭」外觀 |
| 釣魚 | 隨時可釣,結果隨機(可選:用餌 / 體力限制,後期再加) |

### 離線成長與時間權威性

- 作物 / 採集點**離線也會推進**;玩家回到家園時,**由後端用伺服器時間**計算哪些已成熟 / 已復原。
- **時間以伺服器為準**(時間戳存後端),避免玩家改本機時鐘作弊。
- (後期)可加推播:作物成熟時通知手機。

## 家園成長 / 擴建

家園**不是一開始全開**,隨進度逐步解鎖,給生活系統長期目標:

| 解鎖來源 | 可解鎖內容 |
|---|---|
| boss rush 里程碑(首次過第 5 / 10 / 15 / 20 關) | 新區域、新設施、新採集點類型 |
| 花材料 / 貨幣在房子升級 | 更多農田格、縮短成長時間、更好的釣點、倉庫擴容 |

> 形成「打更深 → 解鎖更多生活內容 → 產更多材料 → 裝備更強 → 打更深」的長期循環。

---

## 倉庫 vs 攜帶上限

- **材料 / 道具**存於房子**倉庫**(容量大,可升級)。
- 出戰只能**攜帶有限補血道具**(例如 3~5 格,可升級),製造取捨。
- 裝備在房子更換 / 強化後,進戰鬥才生效(屬性由後端彙整,見 [equipment.md](equipment.md))。

---

## 存檔

- 家園是**安全存檔點**:生產 / 合成 / 強化 / 裝備變更等動作即時存後端。
- 出戰後的單局狀態存 `run_progress`;回到家園代表一局結束。

## 資料模型對應(見 [architecture.md](architecture.md))

| 表 | 與家園相關欄位 |
|---|---|
| `farm_plot` | `plot_index, crop_item_id, planted_at, ready_at` |
| `life_activity` | `activity_type, cooldown_until`(採集點復原 / 釣魚冷卻) |
| `inventory` | 倉庫內容 |
| `player` | 解鎖進度 / 家園升級狀態(可加 `hub_unlocks(jsonb)`) |

> 真實時間模型下,`ready_at` / `cooldown_until` 存**伺服器時間戳**,讀取時後端比對 `now` 判定狀態。
